Sentinel 2 Data Downloader¶

What This Tool Does¶

This program utilizes 2 libraries to download Sentinel 2 optical data, one used to query the Copernicus API, and the other to download the results from the AWS S3 bucket. After the download process, the downloaded data can optionally be atmospherically corrected. The tool will convert the .jp2 band rasters to .tif, and simplify the file naming and folder structure.

Example¶

> python sentinel2dd.py -p .\\shapefiles\\SK_Agr_Mask_WGS84.shp -c 55 -ac 6
0 -d 20170315 20170615

Required Arguments

Command Purpose
-p [polygon_filename] vector polygon filename relative to script,
representing geographic extent to query
-sp [simple_poly_filename] simple vector polygon with a single polygon and < 200 vertices
OR  
-tiles [UUID…] Download specific tiles by UUID, separate each tile UUID
by a space

Optional Arguments

Command Purpose
-d [YYYYMMDD YYYYMMDD] the date range to query by, start end
-c [0 - 100] the percent of total cloud cover to query by,
50 would be 50 or less
-ac [10, 20, 60] 60 the resolution to perform atmospheric correction at,
options are 10, 20, 60 meters
-conv [flag] Specify whether or not to perform conversion to .tif

Building¶

A make file is used to simplify common tasks. Make commands include:

  • make docs
    Runs the sphinx autodoc commands and builds the html folder from the generated .rst.
  • make init
    Installs the required pip packages. Should create and activate a virtualenv before running this.
  • make runexample-big
    Runs the main script with example arguments. See the Makefile for further targets that start with runexample.
  • make clean-all
    After running an example, this will clear out all the folders created by the script. DO NOT run this if data is required and not for testing.

Misc Files¶

Additional config files and files that are useful are stored in the root of the project.

Logging¶

The logging config file is stored as logging.conf.json

It is read by the script when it first runs, and sets up the logging module. Logging is setup to log to default.log file, but this can be changed. By default everything goes to the log file, while only INFO and higher go to the std out.

Product Attributes¶

A sample sentinel2 product dictionary that is returned from the Copernicus API is stored in product_attribute_list.txt. This file can be useful for determining the key for the specific attribute you are trying to access.

Sentinel2 Scene Classifcation Style¶

The file sentinel2classificationstyle.txt stores a Qgis classification style that matches the scene classification specification output from the sen2cor tool.
